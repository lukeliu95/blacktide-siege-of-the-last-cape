# Round 6 · 升级方案 + 开发计划(会议文案)
> 2026-06-20 · Alan 主持 · 多 sub Agent 并行执行

## 一、用户需求(原话拆解)
1. **枪浮动更夸张** — 当前全向跟随幅度偏小,要更明显的晃动感。
2. **弹匣 180 发** — 现 30 发太少,改 180(重机枪火力感)。
3. **解锁视频能力 → 电影级过场动画** — 用 pippit-tool-cli(XYQ)生成开场 CG。
4. **背景音乐** — 加 BGM,氛围拉满。
5. **丧尸兵动画化** — 让走来的丧尸看起来像在动。试两条路:
   - (a) 先做成视频 → 转 GIF/序列帧 → Phaser 帧动画
   - (b) 多视角/多帧贴图拼动画
6. **方式**:完整思考 → 开会出文案 → 开发计划 → 多 sub Agent 分头行动。

## 二、第一性思考 + 取舍(递弱代偿)
| 项 | 决策 | 理由 / 代偿 |
|---|---|---|
| 枪浮动 | 加大跟随系数 + 叠加 idle 呼吸晃动(常驻低频 sin) | 跟随幅度 + 待机微晃 = 更"手持感";过大易晕,留可调参数 |
| 180 发 | clip=180,HUD 弹药从「30 格点」改「数字 + 进度条」 | 180 个点画不下;条形更读得清 |
| 过场视频 | pippit image/text-to-video 出 8s CG → mp4 → title 全屏 `<video>` | 真电影级;风险=安全审核(沿用"虚构怪物/无血"措辞)+ 时长/质量 |
| BGM | **程序化 WebAudio 氛围乐**(低频战鼓 drone + 风声 + 远方炮声)为可靠基线;过场视频自带音轨作开场 | 文/图生视频的"音乐"不稳定;程序化 loop 可靠、免版权、零依赖、可循环 |
| 丧尸动画 | **主攻 (a) 图生视频走路循环 → ffmpeg 抽帧 → 绿幕抠图 → 序列帧 → Phaser anim**;先在 soldier+brute 两类验证管线 | 真走路动画 > 程序化步态;风险=视频帧角色漂移/背景不纯。**先验证 1-2 类再放量**(Musk:验证再规模化) |

## 四、验收
- 枪浮动明显更夸张(待机晃 + 跟随幅度↑);弹匣 180,HUD 正确显示。
- 开场播放电影级过场 CG(可跳过)。
- 进游戏有循环 BGM。
- 至少 1 类丧尸用真·走路序列帧动画(若管线验证通过则放量到 4 类;不通过则诚实回退程序化步态 + 记录原因)。
- 全程 Playwright 实测,控制台仅 favicon 404。

## 五、风险 / 回退
- 视频安全审核驳回 → 软化措辞重试(已有经验)。
- 视频帧角色漂移/背景不纯导致序列帧抖 → 回退当前程序化步态(已可用),如实记录。
- 视频生成超时/失败 → 过场降级为静态 CG 图(GPT-Image)。
