# Intent Brief · 诺曼底碉堡

> Alan Intake · 2026-06-20 · 单回合收敛(用户给出明确画面感 + 我给推荐默认)

## Primary Goal
一个**单文件 HTML5 Canvas 像素射击游戏**:艾森豪威尔站在诺曼底碉堡上(固定机位),海里一波波涌出生化士兵向防线逼近,玩家鼠标瞄准开火,扛住无尽波次。打开浏览器即玩,零依赖、零安装。

## Secondary
- 街机式紧张感:弹匣 + 换弹 + 连击 + 波间空投补给
- 诺曼底战场氛围:灰蓝海、混凝土碉堡、沙袋、捷克刺猬、探照灯、五星旗
- 本地最高分(localStorage)

## 砍掉的伪需求
- ❌ 横向卷轴大地图 / 跑图(用户描述是"站碉堡上打海里的敌人"= 固定机位波次防守)
- ❌ 后端 / 联机 / 账号 / 付费
- ❌ 外部美术资源(全程程序化 Canvas 绘制)

## 硬约束
- 单文件 `index.html`,纯 vanilla JS + Canvas 2D,无构建步骤
- 桌面鼠标为主,兼容触屏

## 钉死的 P0(均按 Alan 推荐拍板)
1. 操作:鼠标瞄准 + 点击/按住开火,A/D 移动,R 换弹,P 暂停
2. 核心循环:无尽波次 + 计分 + 碉堡血量,血空 = Game Over
3. 美术:程序化像素风,诺曼底灰蓝调,Q 版艾森豪威尔(M1 钢盔 + 五星)

## 验收草稿
- 浏览器打开即出标题屏,点击开始进入战斗
- 敌人从海面涌出并逼近碉堡;命中扣血、击杀计分、逼近碉堡掉血
- 弹药/换弹/连击/空投补给/波次推进/Game Over/最高分 全链路可用
- 无控制台报错(favicon 404 除外)

## 派活预案
单一自包含前端工件,Alan 直接构建(Musk 第一性删掉重流水线),前端实现 + Playwright 自测 + impeccable 设计闸门。
